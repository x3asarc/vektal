## RISKS_AND_MITIGATIONS

**Risk 1: Polaris Design System Conflicts**
- **Issue**: Shopify embedded apps typically use Polaris; custom tokens may clash with Polaris CSS variables
- **Mitigation**: Scope custom tokens under `--productops-*` namespace; use CSS containment (`contain: layout style paint`) on root components; test in local Shopify CLI tunnel (`shopify app dev`) to verify no style leakage

**Risk 2: Multi-Supplier Data Density**
- **Issue**: SupplierCard grid may overflow on smaller screens with 10+ suppliers
- **Mitigation**: Implement virtualized list (react-window) in Phase 4; set max-height on dashboard container; horizontal scroll on OrderSummary table as last resort (violation avoided via card-based mobile layout)

**Risk 3: Token Update Cascade**
- **Issue**: Primary color change affects 50+ existing components outside scope
- **Mitigation**: Aura query identified blast radius (`frontend/app/components/legacy/`); implement tokens as CSS layers (`@layer productops`) to ensure specificity wins; provide fallback values in tokens.css

**Risk 4: Accessibility Regression in Interactions**
- **Issue**: Modal transitions may trap focus or disable screen reader announcements
- **Mitigation**: Use `react-focus-lock` and `react-remove-scroll` packages; validate with VoiceOver (macOS) and NVDA (Windows) locally before marking complete; include `aria-live="polite"` regions for notification banners

**Risk 5: Visual Loop Budget Exhaustion**
- **Issue**: Visual satisfaction stuck at 7/10 after 4 loops (budget exceeded)
- **Mitigation**: Pre-validate atoms in Storybook isolation before molecular assembly; define "good enough" criteria (pixel-perfect not required, consistent spacing required); escalate to Commander if architectural constraints block 8/10 score
