## Decisions Made

**1. Catalog-first intelligence approach**
- 50+ products: High confidence, no questionnaire needed
- 10-49 products: Medium confidence, questionnaire to fill gaps
- <10 products: Low confidence, questionnaire is primary source
- Rationale: Large catalogs provide reliable intelligence; small catalogs need user input

**2. TF-IDF for keyword extraction**
- sklearn TfidfVectorizer with German stop words
- Token pattern includes äöüÄÖÜß for German text
- min_df=2 (must appear in 2+ products), max_df=0.95 (ignore too common)
- Rationale: Statistically meaningful keywords, not just word frequency

**3. Pattern learning from SKU examples**
- Pre-defined pattern templates (letter+digit, hyphenated, numeric)
- Match templates against existing SKUs, require min_occurrences=5
- Associate patterns with vendors via frequency
- Rationale: Inferred patterns more reliable than user-provided regex

**4. Niche detection via keyword scoring**
- Score each niche by keyword frequency in titles/tags/types
- Boost confidence if winner is 2x second-best
- Return "other" if no clear match
- Rationale: Multiple signals more reliable than single field
