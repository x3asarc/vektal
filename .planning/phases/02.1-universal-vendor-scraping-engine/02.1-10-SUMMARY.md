## Decisions Made

**Firecrawl for collection crawling:**
- Chosen over manual Playwright scraping for reliability and maintained infrastructure
- Firecrawl handles JavaScript rendering, rate limiting, and pagination automatically
- Async job polling enables large collection crawls without blocking

**Non-destructive mapping merge:**
- GSDPopulator never overwrites existing mappings to preserve manual corrections
- New mappings added incrementally, existing URLs stay untouched
- Prevents data loss from re-crawling same collection

**SKU extraction strategy:**
- Try URL patterns first (fastest, no API calls)
- Fall back to page content scraping if URL doesn't contain SKU
- Multiple pattern support for vendor flexibility
