## Decisions Made

**1. Priority-based pattern matching**
- Specific commands (help, status, list) checked before generic SKU pattern
- Prevents "help" from matching bare SKU pattern
- Ensures 95% confidence for pattern matches

**2. 3-tier classification approach**
- Tier 1: Pattern matching (80%, free, instant)
- Tier 2: Local LLM with sentence-transformers (15%, free, <100ms)
- Tier 3: API LLM with OpenRouter (5%, paid, 1-2s)
- Optimizes cost/latency tradeoff

**3. Structured responses without LLM**
- Handlers generate responses from data, not LLM generation
- Responses are JSON-ready for API consumption
- No LLM overhead for response formatting

**4. Backend-only scope**
- No frontend UI components in this phase
- Phase 5 will expose via REST API (POST /api/chat/message)
- Phase 10 will build ChatGPT-style React UI
