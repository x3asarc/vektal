  <files>src/core/discovery/firecrawl_client.py, src/core/discovery/__init__.py</files>
  <action>
Create `src/core/discovery/firecrawl_client.py` with FirecrawlClient class:

1. **FirecrawlClient class**:
   - Initialize with API key from environment (FIRECRAWL_API_KEY)
   - Base URL: https://api.firecrawl.dev/v1
   - Include user-agent header

2. **crawl() method**:
   - Input: collection page URL, max_pages (default 100)
   - Make POST request to /crawl endpoint
   - Poll for completion (crawl is async)
   - Return list of crawled page URLs and content
   - Handle rate limits with exponential backoff

3. **scrape() method**:
   - Input: single URL
   - Make POST request to /scrape endpoint
   - Return extracted content (markdown, HTML, metadata)
   - Useful for individual product pages

4. **extract_product_urls() method**:
   - Input: crawl result
   - Parse crawled pages to find product URLs
   - Filter to only product pages (not category, about, etc.)
   - Return list of (url, extracted_sku) tuples

5. **Error handling**:
   - FirecrawlAPIError for API errors
   - Rate limit detection and retry
   - Timeout handling for long crawls

6. **Configuration**:
   - Read FIRECRAWL_API_KEY from os.environ
   - Raise clear error if key missing
   - Optional: Accept key in constructor for testing

Update `src/core/discovery/__init__.py` to export FirecrawlClient.

API Reference (from Firecrawl docs):
- POST /v1/crawl: Start crawl job, returns job_id
- GET /v1/crawl/{job_id}: Poll for status and results
- POST /v1/scrape: Scrape single page immediately

Create `src/core/discovery/gsd_populator.py` with GSDPopulator class:

1. **GSDPopulator class**:
   - Takes vendor_name and FirecrawlClient instance
   - Loads existing vendor config using config/loader.py

2. **populate_from_collection() method**:
   - Input: collection_url
   - Call firecrawl_client.crawl(collection_url)
   - Extract product URLs from results
   - Match URLs to SKUs using vendor's sku_patterns
   - Return dict of {sku: direct_url}

3. **_extract_sku_from_url() method**:
   - Input: product URL, vendor sku_patterns
   - Try each SKU pattern regex against URL
   - Extract SKU from URL slug/path
   - Handle variations (lowercase, hyphenated)
   - Return normalized SKU or None

4. **_extract_sku_from_content() method**:
   - Input: page content (from Firecrawl scrape)
   - Look for SKU in common locations:
     - Title element
     - SKU/product code field
     - URL path component
   - Return extracted SKU

5. **update_vendor_config() method**:
   - Input: dict of {sku: url} mappings
   - Load vendor config
   - Merge new mappings into gsd_mappings.sku_to_url
   - Update gsd_mappings.last_updated timestamp
   - Save using save_vendor_config()
   - Return count of new mappings added

6. **verify_mappings() method**:
   - Sample N random mappings
   - Verify URLs still resolve (HEAD request)
   - Return verification stats
