<task type="auto">
  <name>Task 2: Implement selector validation and scoring</name>
  <files>src/core/discovery/site_recon.py</files>
  <action>
Enhance SiteReconnaissance with selector validation:

1. **_score_selector() method**:
   - Input: selector string, expected content type (text, image, price)
   - Score criteria:
     - Element exists on page: +1
     - Multiple elements found (bad for unique fields): -0.5
     - Content matches expected type (numbers for price, URLs for image): +0.5
     - Content is not empty: +0.5
   - Return score 0.0-2.0

2. **_validate_selectors_on_samples() method**:
   - Visit 3-5 sample product pages
   - Test each discovered selector
   - Track success rate per selector
   - Flag selectors with <80% success rate as unreliable

3. **_detect_javascript_requirement() method**:
   - Compare initial HTML with rendered HTML (after JS execution)
   - If product content only appears after JS: requires_javascript = True
   - Check for lazy-loading indicators (IntersectionObserver, data-src)

4. **_find_collection_pages() method**:
   - Look for links matching patterns: /collection*, /category*, /products*
   - Verify they contain product listings
   - Return list of discovered collection URLs

5. **Integration with discover()**:
   - Call validation methods during discovery
   - Add confidence score to returned SiteReconData
   - Include validation_warnings in output
