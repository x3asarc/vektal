## Decisions Made

1. **Pydantic v2 over v1:** Used Pydantic v2 field_validator and model_validator decorators for better validation control
2. **_meta field alias:** Pydantic doesn't allow leading underscores in field names, so used `meta: VendorMeta = Field(alias="_meta")` to maintain YAML compatibility
3. **Early regex validation:** SKU pattern regex validated at schema level rather than runtime to catch configuration errors immediately
4. **URL placeholder validation:** Product URL templates validated to contain {sku} placeholder to prevent misconfiguration
5. **Dict for flexible sections:** Used `dict[str, Any]` for deeply nested sections (selectors, enrichment) to balance type safety with flexibility
