## Decisions Made

**Strategy selection approach:**
- Primary strategy from vendor config, fallback to alternative strategies
- Playwright lazy-imported to avoid requiring installation if unused
- Requests strategy as default fallback

**Retry mechanism:**
- Tenacity library for exponential backoff (2-30s, 3 attempts)
- Rate limit detection via error message patterns (429, "rate limit", "too many")
- RateLimitError exception for retry-specific handling

**Validation approach:**
- Separate validation step after extraction
- Configurable rules from vendor YAML
- SKU normalization with case-insensitive and hyphen-ignoring options
- Placeholder image detection via pattern matching

**Batch processing:**
- asyncio.gather for concurrent scraping within batches
- Configurable batch size (default 10)
- Delay between batches to respect rate limits (default 5000ms)
