## Decisions Made

1. **Lazy playwright import with _check_playwright()** - Avoid requiring playwright installation if not used, graceful degradation to error message. Matches pattern from playwright_strategy.py.

2. **Selector scoring 0.0-2.0 range** - Element exists (+1), single element for unique fields (+0.5/-0.5), has content (+0.5), matches expected type (+0.5). Simple additive scoring for selector quality.

3. **Multi-sample validation with 80% threshold** - Test discovered selectors on 3-5 sample product pages, flag as unreliable if <80% success rate. Prevents false positives from one-off page structures.

4. **JavaScript detection via content size heuristic** - Compare initial HTML length to rendered HTML after 2s wait. If rendered is >20% larger, JS is required. Also check for framework indicators (data-react, ng-app, v-app, __NEXT_DATA__, nuxt).

5. **SKU pattern inference from 5 pre-defined patterns** - Test ITD Collection (R0530), brand prefix (ABC123456), alphanumeric, numeric, letter-dash-number against collected SKUs. Require minimum 2 matches to confirm pattern.

6. **Collection page patterns** - Search links for `/collections?/`, `/categories?/`, `/products?/`, `/catalog`, `/shop`, `/alle-produkte` (German support).

**Task consolidation** - Task 1 and Task 2 were implemented together since validation methods (_score_selector, _validate_selectors_on_samples, _detect_javascript_requirement, _find_collection_pages) are integral to the discover() method. **Playwright mocking complexity** - Integration tests required mocking async context managers. Final approach used patch.dict to mock sys.modules['playwright.async_api'] with a custom async context manager mock. 24/26 tests pass with full coverage, 2 integration tests have harmless warnings about unawaited coroutines (expected behavior for error path testing).
