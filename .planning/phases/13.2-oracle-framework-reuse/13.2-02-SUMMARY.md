## Decisions Made

2. **Async dispatch via .delay()** - All emissions use Celery .delay() to avoid blocking request path
3. **Episode ID from content hash** - Deterministic ID generation from episode_type + store_id + correlation_id + key fields
4. **LRU cache deduplication** - 10k entry LRU cache prevents duplicate episode ingestion
5. **5-second ingestion timeout** - Prevents worker hangs while allowing sufficient time for graph writes
6. **Retry only on transient errors** - Validation errors do not retry; connection/timeout errors retry max 2 times
