## Technical Details

### GraphOracleAdapter

**Query evidence flow:**
1. Check graph availability with timeout
2. Query failure_pattern episodes for target module
3. Analyze failures for critical warnings
4. Return OracleSignal with decision:
   - No failures → `pass` (confidence 0.8)
   - Non-critical failures → `review` (confidence 0.6)
   - Critical failures → `fail` (confidence 0.9, requires_user_action=True)

**Fail-open behavior:**
```python
FAIL_OPEN_SIGNAL = OracleSignal(
    decision='pass',
    confidence=0.5,
    reason_codes=[],
    evidence_refs=[],
    requires_user_action=False,
    source='graph_unavailable'
)
```

### Memory Retrieval

**Retrieval modes:**
- `vector`: Embedding similarity only (placeholder for future implementation)
- `lexical`: Keyword matching with token overlap scoring
- `blend`: Combine vector + lexical with 0.7/0.3 weighting (default)

**Blended scoring algorithm:**
1. Normalize vector scores to [0, 1]
2. Normalize lexical scores to [0, 1]
3. Combine: `combined_score = 0.7 * vector + 0.3 * lexical`
4. Deduplicate by content prefix
5. Sort by combined score descending

**Graph evidence integration:**
- Optional via `include_graph=True` parameter
- Queries graph for relevant episodes
- Adds graph-derived memories to result set
- Marks source='graph' for attribution
- Fail-open: continues without graph results on error
