## Key Decisions Made

### 1. Decision vocabulary harmonization

**Mapping:**
- `accept` → `pass` (clear automatic approval)
- `suggest` → `review` (requires human review but not blocking)
- `hold` → `hold` (unchanged, pending more information)
- `reject` → `fail` (clear rejection/blocking)

**Rationale:** Graph adapter was already using pass/fail/review vocabulary. Harmonizing to this vocabulary instead of the enrichment vocabulary (accept/suggest/reject) provides clearer semantics and better alignment with verification patterns.

### 2. Source field added to OracleDecision

**Change:** Added `source: str = 'enrichment'` field to track decision origin.

**Rationale:** Enables tracking which adapter made the decision (enrichment oracle vs graph oracle vs future adapters). Useful for debugging and telemetry.

### 3. Backward-compat alias instead of migration period

**Approach:** Created `OracleSignal = OracleDecision` alias instead of gradual deprecation.

**Rationale:** Zero external callers detected in codebase. Alias provides insurance for any missed references while unblocking Phase 14/15 immediately.

## Technical Notes

### Contract Fields Comparison

**Before (OracleSignal):**
```python
@dataclass(frozen=True)
class OracleSignal:
    decision: str  # 'pass', 'fail', 'review'
    confidence: float
    reason_codes: List[str]
    evidence_refs: List[str]
    requires_user_action: bool = False
    source: str = 'graph'
```

**After (Unified OracleDecision):**
```python
@dataclass(frozen=True)
class OracleDecision:
    decision: Literal["pass", "review", "hold", "fail"]
    confidence: float
    reason_codes: tuple[str, ...]
    evidence_refs: tuple[str, ...]
    requires_user_action: bool
    source: str = 'enrichment'
```

**Key differences:**
- Decision type is now typed literal (not generic str)
- reason_codes and evidence_refs are tuples (not lists) for immutability
- source field default differs by adapter (enrichment vs graph)

### Legacy Vocabulary Normalization

Added helper for external callers:
```python
def normalize_legacy_decision(decision: str) -> str:
    """Map legacy vocabulary to unified contract."""
    legacy_map = {
        "accept": "pass",
        "suggest": "review",
        "reject": "fail",
    }
    return legacy_map.get(decision, decision)
```
