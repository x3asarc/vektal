  <name>Task 2: Upgrade memory retrieval with vector + lexical blend</name>
  <files>
    src/assistant/memory_retrieval.py
  </files>
  <action>
1. If src/assistant/memory_retrieval.py exists, extend it. If not, create it.

2. Core retrieval function:
   ```python
   def retrieve_relevant_memories(
       query: str,
       store_id: str,
       limit: int = 10,
       include_graph: bool = True,
       retrieval_mode: str = 'blend'  # 'vector', 'lexical', 'blend'
   ) -> List[MemoryResult]:
   ```

3. Retrieval modes:
   a) 'vector': Use embedding similarity only
      - Compute query embedding
      - Search memory facts by cosine similarity
      - Return top-k results

   b) 'lexical': Use keyword matching only
      - Tokenize query
      - Search memory facts by keyword overlap
      - Return top-k results
      - This is the FALLBACK when vector search fails

   c) 'blend' (default): Combine vector + lexical
      - Get vector results (up to 2*limit)
      - Get lexical results (up to 2*limit)
      - Merge and re-rank by combined score:
        combined_score = 0.7 * vector_score + 0.3 * lexical_score
      - Return top limit results
      - On vector failure, fall back to lexical only

4. Graph evidence integration (when include_graph=True):
   - Query GraphOracleAdapter for relevant episodes
   - Add episode summaries to memory results
   - Mark source='graph' for graph-derived memories

5. MemoryResult dataclass:
   ```python
   @dataclass
   class MemoryResult:
       content: str
       relevance_score: float
       source: str  # 'facts', 'graph', 'lexical'
       metadata: dict
   ```

6. Error handling:
   - Vector search failure -> fall back to lexical
   - Lexical search failure -> return empty list
   - Graph query failure -> continue without graph results
   - Log all failures at warning level

Lexical fallback ensures retrieval works even when embedding models unavailable.
