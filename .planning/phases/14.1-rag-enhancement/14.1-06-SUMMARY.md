# Phase 14.1 Plan 06 - Session Lifecycle Hooks and Convention Guardrails

## Summary

Implemented lifecycle-aware convention context and guardrails across query/MCP paths: session startup convention preload, architectural convention checks, and CALLS-based refactor safety scoring.

## What Was Built

### 1. Convention checker module
- Added `src/graph/convention_checker.py`:
  - `ConventionViolation` structured advisory model
  - `load_default_conventions(limit=...)` from `AGENTS.md` + `STANDARDS.md`
  - `check_against_conventions(proposed_change, conventions, threshold)` with deterministic similarity scoring

### 2. CALLS-based refactor guard
- Added `src/graph/refactor_guard.py`:
  - `RefactorSafetyReport`
  - `verify_refactor_safety(file_path)` flow:
    - discovers functions in file
    - queries `function_callers` + `function_callees`
    - computes `risk_level` via thresholds: low `0-2`, medium `3-10`, high `11+`

### 3. Lifecycle hooks in MCP server
- Updated `src/graph/mcp_server.py`:
  - session cache `_SESSION_CONTEXT`
  - `initialize_session_context()` preloads top conventions once per session (fallback to default conventions)
  - first tool response includes `system_context.conventions`
  - `retrieve_intent_tool()` includes convention context + convention violation metadata for architectural queries

### 4. Query hook ordering + convention metadata
- Updated `src/graph/query_interface.py`:
  - retains cache-first query order
  - adds architectural-query detection and convention guardrail checks before returning architectural suggestions
  - populates `conventions_checked` on `QueryResult`

### 5. Template support for lifecycle/refactor checks
- Updated `src/graph/query_templates.py`:
  - `functions_in_file`
  - `top_conventions`

### 6. Unit tests
- Added:
  - `tests/unit/test_convention_checker.py`
  - `tests/unit/test_refactor_guard.py`
- Updated:
  - `tests/unit/test_query_interface.py`
  - `tests/unit/test_mcp_server_contract.py`
