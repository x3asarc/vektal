# Phase 14.1 Plan 02 - Reasoning Provenance and Discrepancy Tracking

## Summary

Implemented provenance capture for graph queries and added discrepancy flagging when graph responses are empty but filesystem evidence exists.

## What Was Built

- Updated `src/core/synthex_entities.py`:
  - Added `ReasoningTraceEntity`
  - Extended `EpisodeType` with:
    - `query_reasoning_trace`

### 2. Episode ID support for new event families
- Updated `src/tasks/graphiti_sync.py`:

    - `query_reasoning_trace`
    - `graph_discrepancy`

### 3. Query trace emission + discrepancy detection
- Updated `src/graph/query_interface.py`:
  - Added `source` and `discrepancy_flagged` fields to `QueryResult`

  - Added filesystem fallback path detection for graph-null scenarios

## Battle-Test Notes

- Query trace emission is non-blocking: episode emission errors are logged at debug and do not block query responses.
- Discrepancy path is deterministic for explicit `file_path` template queries and path-like query tokens.

## Learnings

1. Adding optional metadata-return paths in shared helpers (`query_with_fallback`) preserves compatibility while enabling richer observability for new consumers.
2. Query-level discrepancy logic is easiest to validate when fallback detection starts with explicit `file_path` semantics before broader filesystem search heuristics.
