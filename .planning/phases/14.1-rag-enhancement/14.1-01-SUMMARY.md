# Phase 14.1 Plan 01 - Memory Entity Types and Relationships

## Summary

Implemented the Phase `14.1-01` memory foundation by extending graph entity contracts, schema indexes/constraints, deterministic episode IDs, and adding a dry-run memory seeding script.

## What Was Built

### 1. Memory entities + edges
- Updated `src/core/synthex_entities.py`:
  - Added `DecisionEntity`
  - Added `ConventionEntity`
  - Added `BugRootCauseEntity`
  - Added `ExplainsEdge`
  - Added `ResolvedByEdge`
  - Added `SupersedesEdge`
  - Extended `EpisodeType` with:
    - `decision_recorded`
    - `convention_established`
    - `bug_root_cause_identified`

### 2. Schema support for memory nodes
- Updated `src/core/codebase_schema.py`:
  - Added unique constraints/indexes for `Decision`, `Convention`, `BugRootCause`
  - Added query indexes for `Convention.scope` and `Decision.status`
  - Added required field constraints for `Decision.title` and `Convention.rule`

### 3. Deterministic IDs for new episode families
- Updated `src/tasks/graphiti_sync.py`:
  - Extended `_generate_episode_id()` branching for:
    - `decision_recorded`
    - `convention_established`
    - `bug_root_cause_identified`

### 4. Memory node seeder with dry-run
- Added `scripts/graph/seed_memory_nodes.py`:
  - Supports `--dry-run`
  - Extracts candidate conventions/decisions from project docs
  - Uses idempotent `MERGE` writes when graph is available
  - Fail-open behavior if graph client is unavailable

### 5. Test coverage updates
- Updated `tests/core/test_synthex_entities.py`:
  - Added entity tests for Decision/Convention/BugRootCause
  - Added edge tests for Explains/ResolvedBy/Supersedes
  - Updated enum assertions for new episode types
- Updated `tests/tasks/test_graphiti_sync_contract.py`:
  - Added deterministic ID coverage for new memory episode type payloads
