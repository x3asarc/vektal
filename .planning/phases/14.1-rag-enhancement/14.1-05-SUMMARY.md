# Phase 14.1 Plan 05 - MCP Tool Server

## Summary

Implemented the MCP server surface for graph access with three tools (`query_graph`, `get_dependencies`, `retrieve_intent`), pinned MCP dependency, and added local Claude MCP registration.

## What Was Built

### 1. MCP server module
- Added `src/graph/mcp_server.py`:
  - Tool contracts and handlers:
    - `query_graph_tool(query)`
    - `get_dependencies_tool(file_path, direction, depth)`
    - `retrieve_intent_tool(query)`
  - `list_tool_contracts()` for explicit contract visibility
  - stdio MCP server bootstrap (`run_server()`) using low-level MCP SDK when available
  - graceful import behavior when `mcp` dependency is missing

### 2. Startup wrapper
- Added `scripts/graph/start_mcp_server.sh`:
  - project-root + `PYTHONPATH` setup
  - graph availability pre-check
  - stdio server launch command
  - `--help` support

### 3. Dependency pinning
- Updated `requirements.txt`:
  - added exact pin `mcp==1.12.4`

### 4. Claude local settings integration
- Updated `.claude/settings.local.json`:
  - added top-level `mcpServers` entry:
    - `synthex-knowledge-graph` -> `python src/graph/mcp_server.py`
  - preserved existing `permissions` block

### 5. MCP contract tests
- Added `tests/unit/test_mcp_server_contract.py`:
  - tool list contains all expected tools
  - query tool output contract
  - dependency tool dedupe/impact contract
  - intent tool confidence/source contract

## Battle-Test Notes

- Direct shell execution of `bash scripts/graph/start_mcp_server.sh --help` is blocked in this environment (`E_ACCESSDENIED`), so runtime behavior was validated via module-level tests and script structure checks.

## Learnings

1. Keep MCP tool handlers callable as plain Python functions; this makes contract testing independent of transport/runtime.
2. MCP server config must be additive in `.claude/settings.local.json` to avoid clobbering existing permission controls.
