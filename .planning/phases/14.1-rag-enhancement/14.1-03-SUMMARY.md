# Phase 14.1 Plan 03 - Search-then-Expand Bridge

## Summary

Implemented a bounded Search-then-Expand bridge that seeds context with vector similarity and expands related graph context under strict limits.

## What Was Built

### 1. Bridge module
- Added `src/graph/search_expand_bridge.py`:
  - `BridgeConfig` with defaults:
    - `max_initial_nodes=5`
    - `max_traversal_depth=2`
    - `max_context_tokens=8192`
    - relationship allowlist (`IMPORTS`, `CALLS`, `IMPLEMENTS`, `EXPLAINS`)
  - `BridgeResult` with:
    - `initial_nodes`
    - `expanded_nodes`
    - `total_tokens_estimated`
    - `truncated`
    - `relationships_followed`
  - `search_then_expand()` orchestration with:
    - vector seed
    - bounded expansion
    - deduplication
    - token-budget truncation

### 2. Query interface integration
- Updated `src/graph/query_interface.py`:
  - Added `query_with_bridge(query_text)`
  - Unmatched natural-language queries now route to bridge
  - Bridge path returns `source="bridge"` and `query_type="natural_language"`

### 3. Bridge test coverage
- Added `tests/unit/test_search_expand_bridge.py`:
  - max initial node cap test
  - token budget truncation test
  - deduplication across initial/expanded sets test
- Updated `tests/unit/test_query_interface.py`:
  - Added unmatched-query bridge routing test

## Battle-Test Notes

- Bridge tests were hardened to avoid accidental real embedding-model loads by injecting synthetic query embeddings.
- This keeps test runtime stable and validates bridge logic in isolation.

## Learnings

1. Retrieval orchestration tests should isolate model loading to avoid hidden latency and flakiness.
2. Token-budget and dedupe assertions are the highest-value invariants for preventing bridge regressions.
