  <action>
Create .gitattributes file in project root with line ending normalization rules.

Content must include:
```
# Force LF line endings for all text files (critical for Docker on Windows)
# Prevents "bad interpreter" errors in shell scripts inside containers
* text=auto eol=lf

# Explicitly mark script and config files
*.sh text eol=lf
*.py text eol=lf
*.yml text eol=lf
*.yaml text eol=lf
Dockerfile* text eol=lf
*.conf text eol=lf
*.md text eol=lf
*.json text eol=lf
*.env* text eol=lf

# Binary files - do not modify
*.png binary
*.jpg binary
*.jpeg binary
*.gif binary
*.ico binary
*.db binary
*.sqlite binary
```

After creating, run `git add .gitattributes` to ensure it takes effect.

Why: Windows Git converts LF to CRLF by default. Docker containers expect Unix LF. This prevents "/bin/bash^M: bad interpreter" errors.

**IMPORTANT: Single-stage Dockerfile** - Per CONTEXT.md, multi-stage builds are deferred to Phase 13 for optimization. This is a simple, development-focused Dockerfile.

**Dockerfile.backend content:**
```dockerfile
# Flask backend container
# Base image: Debian-slim (not Alpine) for better Python wheel compatibility
# Note: Single-stage build for development simplicity. Multi-stage optimization in Phase 13.
FROM python:3.12-slim

# Create non-root user for security
RUN useradd -m -u 1000 appuser

WORKDIR /app

# Install system dependencies for Python packages
# Some packages (Pillow, lxml) need these build tools
RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Copy and install Python dependencies first (layer caching)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Install Gunicorn and Celery (not in requirements.txt yet)
RUN pip install --no-cache-dir gunicorn celery redis psycopg2-binary

# Copy Gunicorn configuration
COPY gunicorn_config.py .

# Copy application code
# Note: In development, this is overridden by bind mounts
COPY src/ ./src/
COPY utils/ ./utils/
COPY config/ ./config/

# Switch to non-root user
USER appuser

# Expose Flask port
EXPOSE 5000

# Default command - development mode uses Flask dev server
# Production (Phase 13) switches to: gunicorn -c gunicorn_config.py src.app:app
CMD ["flask", "run", "--host=0.0.0.0", "--reload"]
```

**gunicorn_config.py content:**
```python
"""
Gunicorn WSGI server configuration
Used in production mode (Phase 13), not during development
"""
import multiprocessing

# Worker configuration
# Formula: (2 * CPU cores) + 1
workers = multiprocessing.cpu_count() * 2 + 1

# Use sync workers (default)
# For I/O-bound async work, consider: worker_class = 'gevent'
worker_class = 'sync'

# Binding
bind = '0.0.0.0:5000'

# Timeout for long-running requests (AI analysis, scraping)
# Default 30s is too short for Vision AI calls
timeout = 120

# Logging to stdout/stderr for Docker
accesslog = '-'
errorlog = '-'
loglevel = 'info'

# Graceful restart timeout
graceful_timeout = 30

# Maximum requests per worker before restart (memory leak protection)
max_requests = 1000
max_requests_jitter = 50
```

Why python:3.12-slim not Alpine: Alpine uses musl libc which breaks prebuilt wheels for NumPy, Pillow, cryptography. Debian-slim has better compatibility.
Why non-root user: Security best practice - processes share host UID namespace.
