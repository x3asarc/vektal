**IntentRecord dataclass:**
```python
@dataclass
class IntentRecord:
    # What was created/modified
    file_path: str
    entity_type: str  # "file", "function", "class"
    entity_name: str

    # Why it was created (semantic meaning)
    intent: str  # Brief description of purpose
    reasoning: str  # Why this approach was chosen
    alternatives_considered: List[str]  # Other approaches rejected

    # Context
    phase: Optional[str]  # Phase being implemented
    plan: Optional[str]   # Plan being executed
    task: Optional[str]   # Task within plan
    user_request: Optional[str]  # Original user request

    # Metadata
    agent: str  # "claude", "codex", "gemini"
    timestamp: datetime = field(default_factory=datetime.utcnow)
    confidence: float = 1.0  # How confident in approach (0.0-1.0)
```

**capture_intent(intent: IntentRecord) -> None**
- Validate record
- Log locally (for immediate debugging)
- Queue for graph emission (async, non-blocking)

**emit_intent_episode(intent: IntentRecord) -> None**
- Convert IntentRecord to graph episode
- Use existing emit_episode infrastructure from 13.2
- Fire-and-forget (non-blocking)

# Code Generator Agent

Agent for generating code with intent capture for knowledge graph enrichment.

## Intent Capture

When generating code, capture intent for the knowledge graph:

1. **Before writing code**, formulate intent:
   - What am I creating?
   - Why this approach?
   - What alternatives did I consider?

2. **After writing code**, emit intent:
   ```python
   from src.graph.intent_capture import capture_intent, IntentRecord

   intent = IntentRecord(
       file_path="path/to/file.py",
       entity_type="function",  # or "class", "file"
       entity_name="function_name",
       intent="Brief description of what this does",
       reasoning="Why I chose this approach",
       alternatives_considered=["Alternative A", "Alternative B"],
       phase="14",
       plan="14-07",
       task="Task 1",
       agent="claude"
   )
   capture_intent(intent)
   ```

## When to Capture Intent

- Creating new files
- Adding new functions or classes
- Significant refactoring
- Architectural decisions

## When NOT to Capture

- Trivial changes (typo fixes, formatting)
- Generated boilerplate
- Test files (unless testing is the intent)

## Example

Creating a new parser function:

```python
# Before: Think about intent
# Intent: Parse commit messages to extract phase/plan references
# Reasoning: Using regex patterns because commit format is well-defined
# Alternatives: Could use NLP, but simpler approach sufficient

# After: Emit intent
capture_intent(IntentRecord(
    file_path="src/graph/commit_parser.py",
    entity_type="function",
    entity_name="parse_commit_message",
    intent="Parse commit messages to extract phase/plan references",
    reasoning="Using regex patterns because commit format is well-defined. Conventional commits format (feat/fix/docs) is predictable.",
    alternatives_considered=["NLP-based parsing", "String splitting only"],
    phase="14",
    plan="14-04",
    agent="claude"
))
```
