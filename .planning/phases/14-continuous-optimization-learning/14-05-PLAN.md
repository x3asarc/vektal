**Implementation:**

1. **Parse commit message** for phase/plan references
2. **For each changed file:**
   - Determine file type (Python, Markdown, YAML)
   - Parse file with appropriate parser
   - Create/update FileEntity
   - Extract classes/functions (if Python)
   - Extract planning doc info (if .planning/)
   - Generate embedding for file summary

3. **Update relationships:**
   - IMPORTS edges for Python files
   - CONTAINS edges (module -> file -> class/function)
   - IMPLEMENTS edges (from commit phase/plan)
   - REFERENCES edges (from natural references)

4. **Handle deletions:**
   - Detect files removed in commit
   - Mark entities as deleted (soft delete) or remove edges

5. **Optimization:**
   - Skip unchanged content (compare content_hash)
   - Batch graph writes
   - Fail-open if graph unavailable

- Use git diff --cached --name-only
- Return list of files staged for commit

<task type="auto">
  <name>Task 2: Create pre-commit hook</name>
  <files>scripts/hooks/pre-commit-graph-sync.py</files>
  <action>
Create pre-commit hook script for graph synchronization.

**scripts/hooks/pre-commit-graph-sync.py:**
```python
#!/usr/bin/env python3
"""
Pre-commit hook for codebase knowledge graph synchronization.

Runs on every commit:
1. Gets list of staged files
2. Parses commit message for phase/plan references
3. Updates graph with changed file entities
4. Creates planning doc linkages

Non-blocking: Warns but allows commit if graph unavailable.
"""
```

**Behavior:**

1. **Check if graph sync is enabled:**
   ```python
   if os.environ.get('GRAPH_SYNC_ENABLED', 'true').lower() != 'true':
       print("Graph sync disabled, skipping")
       sys.exit(0)
   ```

2. **Get staged files:**
   ```bash
   git diff --cached --name-only
   ```

3. **Get commit message (for interactive commits, read from file):**
   ```python
   commit_msg_file = sys.argv[1] if len(sys.argv) > 1 else None
   if commit_msg_file:
       with open(commit_msg_file) as f:
           commit_message = f.read()
   ```

4. **Run incremental sync:**
   - Parse commit message
   - Sync changed files
   - Create planning links

5. **Output:**
   ```
   [graph-sync] Syncing 3 changed files...
   [graph-sync] Created 5 entities, 8 relationships
   [graph-sync] Linked to Phase 14-05
   ```

6. **Fail-open behavior:**
   - If graph unavailable: warn and exit 0
   - If sync errors: warn and exit 0
   - Never block commit

**Exit codes:**
- 0: Success (or graceful failure)
- Non-zero only for script errors (not graph issues)

Configure pre-commit framework to run the graph sync hook.

**Create or update .pre-commit-config.yaml:**
```yaml
repos:
  # Local hooks
  - repo: local
    hooks:
      # Codebase knowledge graph sync
      - id: graph-sync
        name: Sync codebase knowledge graph
        entry: python scripts/hooks/pre-commit-graph-sync.py
        language: python
        pass_filenames: false
        stages: [commit-msg]
        always_run: true
        verbose: true
```

**Alternative: Use commit-msg hook stage** to access commit message:
```yaml
      - id: graph-sync
        name: Sync codebase knowledge graph
        entry: python scripts/hooks/pre-commit-graph-sync.py
        language: python
        stages: [commit-msg]  # Runs after commit message is written
```
