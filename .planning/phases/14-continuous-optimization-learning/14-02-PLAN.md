  <files>src/core/summary_generator.py</files>
  <action>
Create summary generator that extracts hierarchical summaries for embedding. Per CONTEXT.md decision: embed summaries, not full content.

**Summary format per entity type:**

1. **generate_file_summary(file_path: str) -> str**
   ```
   File: {file_path}
   Purpose: {file_docstring or first comment}
   Exports: {comma-separated exported names}
   Imports: {comma-separated import names}
   ```

2. **generate_function_summary(file_path: str, function_name: str, signature: str, docstring: str) -> str**
   ```
   Function: {module}.{class}.{function_name} or {module}.{function_name}
   Signature: {signature}
   Purpose: {docstring first line}
   File: {file_path}
   ```

3. **generate_class_summary(file_path: str, class_name: str, bases: List[str], docstring: str, methods: List[str]) -> str**
   ```
   Class: {module}.{class_name}
   Inherits: {comma-separated bases}
   Purpose: {docstring first line}
   Methods: {comma-separated method names}
   File: {file_path}
   ```

4. **generate_planning_doc_summary(path: str, doc_type: str, goal: str, status: str) -> str**
   ```
   Planning: {path}
   Type: {doc_type}
   Goal: {goal}
   Status: {status}
   ```

Use Python ast module for parsing. Handle:
- Missing docstrings (use "No description available")
- Binary files (skip, return None)
- Parse errors (log warning, return None)
- Non-Python files (extract from comments or frontmatter)

For Markdown files, extract:
- Title from first # heading
- Goal from "Goal:" line if present
- Status from frontmatter if YAML

**Key components:**

   - Same model used in enrichment pipeline (consistency)
   - 23MB download, runs locally

2. **generate_embedding(text: str) -> List[float]**
   - Lazy model loading (load on first use, not import)
   - Cache model instance (singleton pattern)
   - Return 384-dimension float list
   - Handle empty text (return zero vector)

3. **VECTOR_INDEX_CONFIG** - Neo4j vector index configuration:
   ```python
   VECTOR_INDEX_CONFIG = {
       "index_name": "codebase_embeddings",
       "node_label": "CodeEntity",  # Abstract label for all code nodes
       "property_name": "embedding",
       "dimension": 384,
       "similarity_function": "cosine"
   }
   ```

4. **batch_generate_embeddings(texts: List[str]) -> List[List[float]]**
   - Batch processing for efficiency
   - Progress callback optional
   - Handle empty list (return empty list)

5. **Neo4j vector index creation Cypher:**
   ```cypher
   CALL db.index.vector.createNodeIndex(
       'codebase_embeddings',
       'CodeEntity',
       'embedding',
       384,
       'cosine'
   )
   ```

Add create_vector_index(client) function that creates the index if not exists.
