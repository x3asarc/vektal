  <name>Task 1: Create codebase entity models</name>
  <files>src/core/codebase_entities.py</files>
  <action>
Create Pydantic v2 entity models for codebase knowledge graph nodes. Follow the pattern established in synthex_entities.py (BaseEntity inheritance, frozen discriminators).

Entity types to create:
1. **FileEntity** - Represents a source file
   - path: str (relative to project root)
   - language: str (python, typescript, yaml, markdown)
   - purpose: str (extracted from docstring or first comment)
   - exports: List[str] (exported names)
   - line_count: int
   - last_modified: datetime
   - content_hash: str (for change detection)

2. **ModuleEntity** - Python module (directory with __init__.py)
   - path: str
   - name: str
   - purpose: str
   - is_package: bool
   - submodules: List[str]

3. **ClassEntity** - Class definition
   - file_path: str
   - name: str
   - full_name: str (module.ClassName)
   - purpose: str (from docstring)
   - methods: List[str]
   - bases: List[str] (parent classes)
   - line_start: int
   - line_end: int

4. **FunctionEntity** - Function or method
   - file_path: str
   - name: str
   - full_name: str (module.class.function or module.function)
   - signature: str (parameters and return type)
   - purpose: str (from docstring)
   - is_async: bool
   - line_start: int
   - line_end: int

5. **PlanningDocEntity** - Planning document (.planning/)
   - path: str
   - doc_type: str (PLAN, SUMMARY, CONTEXT, VERIFICATION, ROADMAP, STATE)
   - phase_number: Optional[str]
   - plan_number: Optional[str]
   - title: str
   - goal: Optional[str]
   - status: Optional[str]

Use frozen=True on entity_type fields to prevent mutation. Include validator for path normalization (forward slashes, relative paths).

Do NOT embed vectors yet - that's Plan 02.

   - container_path: str

   - caller: str (full_name)
   - callee: str (full_name)
   - call_count: int

4. **InheritsEdge** - Class inherits from another class
   - child_class: str
   - parent_class: str

5. **ImplementsEdge** - Code implements a planning doc requirement
   - code_path: str (file or function)
   - plan_path: str
   - evidence: str (commit message or comment reference)

6. **ReferencesEdge** - Code references a planning doc (comment, docstring)
   - code_path: str
   - doc_path: str
   - reference_type: str (comment, docstring, commit)

Also add CodebaseEdgeType enum with all edge types for consistent usage.

```python
CODEBASE_INDEXES = [
    # Node indexes for fast lookups
    {"label": "File", "property": "path", "type": "UNIQUE"},
    {"label": "Module", "property": "path", "type": "UNIQUE"},
    {"label": "Class", "property": "full_name", "type": "UNIQUE"},
    {"label": "Function", "property": "full_name", "type": "UNIQUE"},
    {"label": "PlanningDoc", "property": "path", "type": "UNIQUE"},
    # Composite indexes for common queries
    {"label": "File", "properties": ["language", "last_modified"]},
    {"label": "Function", "properties": ["file_path", "name"]},
]

CODEBASE_CONSTRAINTS = [
    # Ensure referential integrity
    {"type": "EXISTS", "label": "File", "property": "path"},
    {"type": "EXISTS", "label": "Function", "property": "full_name"},
]
```
