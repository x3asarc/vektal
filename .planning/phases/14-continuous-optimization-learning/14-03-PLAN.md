  <name>Task 2: Create codebase scanner</name>
  <files>src/graph/codebase_scanner.py</files>
  <action>
Create full codebase scanner that indexes all project files.

**Directories to scan (per CONTEXT.md):**
- src/ - all Python modules, classes, functions
- tests/ - track what they test
- .planning/ - phases, plans, requirements (CENTRAL NODES)
- docs/ - architecture docs, guides

**Exclude patterns:**
- __pycache__/, .git/, node_modules/, .venv/, venv/
- *.pyc, *.pyo, *.so, *.dll
- Binary files (images, compiled assets)

**scan_codebase(root_path: str, config: ScanConfig) -> CodebaseScanResult**
```python
@dataclass
class ScanConfig:
    include_dirs: List[str] = field(default_factory=lambda: ["src", "tests", ".planning", "docs"])
    exclude_patterns: List[str] = field(default_factory=lambda: ["__pycache__", ".git", "node_modules"])
    generate_embeddings: bool = True
    checkpoint_file: Optional[str] = None  # For resumability
    batch_size: int = 100  # Files per batch for progress

@dataclass
class CodebaseScanResult:
    files: List[FileEntity]
    modules: List[ModuleEntity]
    classes: List[ClassEntity]
    functions: List[FunctionEntity]
    planning_docs: List[PlanningDocEntity]
    relationships: List[BaseEdge]  # ImportsEdge, ContainsEdge, etc.
    errors: List[str]
    scan_duration_seconds: float
```

**Implementation:**
1. Walk directory tree respecting exclude patterns
2. Parse each file based on language
3. Create entities from parse results
4. Extract relationships (imports, contains)
5. Generate embeddings for summaries (if enabled)
6. Checkpoint progress every batch_size files
7. Resume from checkpoint if provided

**Relationship extraction:**
- IMPORTS: From Python import statements
- CONTAINS: Module -> File -> Class/Function hierarchy
- IMPLEMENTS: Code references to planning docs (from commit messages, comments)
