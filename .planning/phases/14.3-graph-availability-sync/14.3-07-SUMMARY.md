# Phase 14.3-07 Summary: Sentry Integration & Autonomous Self-Healing

## Status: GREEN

Connected external failure signals from Sentry to the autonomous self-healing layer, completing the full observability-to-remediation pipeline.

## Deliverables

1.  **Sentry Issue Puller (`scripts/observability/sentry_issue_puller.py`):**
    *   Implemented REST API polling for unresolved errors.
    *   Normalization logic maps Sentry issues to taxonomy: `AURA_UNREACHABLE`, `LOCAL_NEO4J_START_FAIL`, `SNAPSHOT_CORRUPT`, `SYNC_TIMEOUT`.
    *   Manual mode support for local testing with mock failure injections.

2.  **Sentry Ingestor (`src/graph/sentry_ingestor.py`):**
    *   Appends normalized failure records to `FAILURE_JOURNEY.md` for long-term audit and learning.
    *   Emits `SYSTEM_FAILURE` episodes to the knowledge graph.
    *   Triggers the healing orchestrator for whitelisted categories.

3.  **Healer Orchestrator (`scripts/graph/orchestrate_healers.py`):**
    *   Centralized dispatcher for autonomous remediation tasks.
    *   Implemented a 5-minute cooldown/throttling mechanism per failure category to prevent remediation loops.
    *   Persists throttle state in `.graph/healer-throttle.json`.
