# Phase 14.3-02 Summary: Remediation Registry & Autonomous Tools

## Status: GREEN

Created new remediation tools for Aura, Snapshots, and Sync pipelines. Migrated all infrastructure fixing logic to the centralized `RemediationRegistry` (NullClaw pattern).

## Deliverables

1.  **New Remediators (`src/graph/remediators/`):**
    *   `AuraRemediator`: Handles Aura Cloud resume/status via Console API v1 with 1hr OAuth token caching.
    *   `SnapshotRemediator`: Rebuilds corrupted or stale local graph snapshots from git HEAD using `local_graph_store`.
    *   `SyncRemediator`: Diagnoses sync failures and retries incremental sync with exponential backoff.

2.  **Registry Upgrades (`src/graph/remediation_registry.py`):**
    *   Implemented `auto_discover()` on singleton initialization.
    *   Made discovery inclusive of all `UniversalRemediator` subclasses (not just `*_remediator.py`).
    *   Established the registry as the single source of truth for all healing tools.

3.  **Infrastructure Hardening:**
    *   `src/graph/infra_probe.py`: Added `active_remediation_loop()` which probes and fixes services via registry dispatch.
    *   `src/graph/backend_resolver.py`: Refactored `attempt_docker_start` to use the `docker` remediator tool from the registry.
    *   Fixed legacy bugs in `redis_remediator.py` and `git_staleness_guard.py` to comply with the `UniversalRemediator` contract.
