# Task 14.2-02: Tool Nodes in Neo4j + search_tools Meta-Tool

**Status**: PLANNED
**Wave**: 2 (Infrastructure)
**Depends on**: None (additive schema extension)
**Duration estimate**: 60-90 minutes
**Risk level**: LOW (additive Neo4j schema, new MCP tool)

---

## Goal

Make the tool registry itself **graph-searchable** by storing tool schemas as `ToolNode` entities in Neo4j, then expose a `search_tools` MCP tool that queries the graph to discover tools by natural language intent.

**Why**: Phase 15 self-healing agents will need to discover remediation capabilities dynamically. Deferred loading (14.2-03) requires a discovery mechanism. This is that mechanism.

---

## Background

**Tool Calling 2.0 Pattern**: "Tool Search" - Instead of loading all 15+ tool schemas upfront (consuming 1200+ tokens), expose only a `search_tools` meta-tool. Claude calls it when needed: `search_tools("tools for updating product prices")` → returns only `product_mutation_tool` schema.

**Benefits**:
- Token efficiency: Only load schemas actually needed
- Dynamic capability discovery: Agents can "learn what they can do"
- Version tracking: Tool schema changes tracked as graph updates
- Cross-tier visibility: Query which tools are `ALLOWED_IN` Tier 2 vs Tier 3

**Source**: [MCP Best Practices - Tool Discovery](https://modelcontextprotocol.info/docs/best-practices/) - notifications for dynamic tool updates

---

## Current State

**Tool storage**:
- MCP tools: Hardcoded in `src/graph/mcp_server.py::list_tool_contracts()`
- Assistant tools: PostgreSQL `assistant_tool_registry` table

**Discovery**:
- MCP: Client fetches all tools via `list_tools()` (no filtering)
- Assistant: Tier-based filtering in `tool_projection.py`

**Gap**: No semantic search over tool capabilities. No graph representation of tool relationships.

---

## Success Criteria

1. ✅ `ToolNode` entity type exists in Neo4j schema
2. ✅ `REQUIRES_INTEGRATION` edge type (Tool → Integration context)
3. ✅ `ALLOWED_IN` edge type (Tool → Tier constraint)

7. ✅ Schema change invalidation: Updating a tool in code triggers node refresh

### Step 1: Extend Entity Schema (`src/core/synthex_entities.py`)

Add `ToolNode` entity:

```python
# src/core/synthex_entities.py

@dataclass(frozen=True)
class ToolNode(BaseEntity):
    """Represents a tool (MCP or assistant) in the knowledge graph."""
    name: str                    # e.g., "query_graph"
    description: str             # Summary of what tool does
    tool_type: str               # "mcp" | "assistant"
    tier_restriction: int | None # 1, 2, 3, or None (all tiers)
    schema_json: str             # Full JSON schema as string
    schema_hash: str             # SHA256 of schema_json for change detection
    input_examples: str          # JSON array of examples (from 14.2-01)
    last_updated: str            # ISO timestamp

# Edge types
@dataclass(frozen=True)
class RequiresIntegrationEdge(BaseEdge):
    """Tool → Integration context (e.g., query_graph requires Neo4j)."""
    integration_name: str        # "neo4j" | "postgresql" | "openrouter"

@dataclass(frozen=True)
class AllowedInEdge(BaseEdge):
    """Tool → Tier constraint (e.g., mutation tools only in Tier 2+)."""
    tier: int                    # 1, 2, or 3
```

### Step 2: Add Cypher Query Template (`src/graph/query_templates.py`)

Add `tool_search` template:

```python
# src/graph/query_templates.py

QUERY_TEMPLATES = {
    # ... existing templates ...

    "tool_search": """
        // Semantic search over tool descriptions + vector similarity
        CALL db.index.vector.queryNodes(
            'tool_embedding_index',
            $top_k,
            $query_embedding
        ) YIELD node, score
        WHERE node:Tool
        AND ($tier IS NULL OR EXISTS {
            MATCH (node)-[:ALLOWED_IN]->(t:Tier {level: $tier})
        })
        RETURN node.name AS name,
               node.description AS description,
               node.schema_json AS schema,
               node.input_examples AS examples,
               score
        ORDER BY score DESC
        LIMIT $top_k
    """
}
```

**Alternative** (if vector index not yet available): Fallback to text-based search:

```python
    "tool_search_text": """
        MATCH (t:Tool)

               t.description AS description,
               t.schema_json AS schema,
               t.input_examples AS examples,
```

```python
# src/graph/mcp_server.py

def list_tool_contracts() -> list[dict]:
    return [
        # ... existing 3 tools ...
        {
            "name": "search_tools",
            "description": "Search for available tools by natural language query. "
                           "Returns tool schemas matching the intent.",
            "inputSchema": {
                "type": "object",
                "properties": {
                    "query": {
                        "type": "string",
                        "description": "Natural language description of what you want to do"
                    },
                    "tier": {
                        "type": "integer",
                        "description": "Optional tier restriction (1, 2, or 3)",
                        "enum": [1, 2, 3]
                    },
                    "top_k": {
                        "type": "integer",
                        "description": "Max number of tools to return",
                        "default": 3
                    }
                },
                "required": ["query"]
            },
            "input_examples": [
                {"query": "tools for updating product prices"},
                {"query": "tools allowed in tier 2 that mutate data", "tier": 2},
                {"query": "read product information", "top_k": 5}
            ]
        }
    ]


async def call_tool(name: str, arguments: dict) -> dict:
    # ... existing tool handlers ...

    if name == "search_tools":
        return await _search_tools_handler(
            query=arguments["query"],
            tier=arguments.get("tier"),
            top_k=arguments.get("top_k", 3)
        )


async def _search_tools_handler(query: str, tier: int | None, top_k: int) -> dict:
    """Search Neo4j for tools matching the query."""
    from src.graph.query_interface import execute_template_query
    from src.core.embeddings import get_embedding  # Assumes embedding function exists

    # Generate query embedding
    query_embedding = get_embedding(query)

    # Execute graph query
    results = await execute_template_query(
        template_name="tool_search",
        params={
            "query_embedding": query_embedding,
            "tier": tier,
            "top_k": top_k
        }
    )

    # Format results
    tools = []
    for record in results:
        tools.append({
            "name": record["name"],
            "description": record["description"],
            "schema": json.loads(record["schema"]),
            "examples": json.loads(record["examples"]) if record["examples"] else [],
            "relevance_score": record["score"]
        })

    return {
        "tools": tools,
        "query": query,
        "tier_filter": tier
    }
```

### Step 4: Tool Node Sync (`src/tasks/incremental_sync.py`)

Add function to sync tool registry → Neo4j:

```python
# src/tasks/incremental_sync.py

def sync_tool_nodes_to_graph():
    """Sync MCP + assistant tools to Neo4j as ToolNode entities."""
    from src.graph.mcp_server import list_tool_contracts
    from src.models.assistant_tool_registry import AssistantToolRegistry
    from src.jobs.graphiti_ingestor import GraphitiIngestor
    import hashlib
    import json

    ingestor = GraphitiIngestor()
    tools_to_sync = []

    # MCP tools
    for tool in list_tool_contracts():
        schema_json = json.dumps(tool, sort_keys=True)
        schema_hash = hashlib.sha256(schema_json.encode()).hexdigest()[:16]

        tool_node = ToolNode(
            name=tool["name"],
            description=tool["description"],
            tool_type="mcp",
            tier_restriction=None,  # MCP tools available to all tiers
            schema_json=schema_json,
            schema_hash=schema_hash,
            input_examples=json.dumps(tool.get("input_examples", [])),
            last_updated=datetime.utcnow().isoformat()
        )
        tools_to_sync.append(tool_node)

    # Assistant tools from PostgreSQL
    assistant_tools = AssistantToolRegistry.query.all()
    for tool in assistant_tools:
        schema_json = json.dumps(tool.metadata_json)
        schema_hash = hashlib.sha256(schema_json.encode()).hexdigest()[:16]

        tool_node = ToolNode(
            name=tool.name,
            description=tool.description,
            tool_type="assistant",
            tier_restriction=tool.min_tier,
            schema_json=schema_json,
            schema_hash=schema_hash,
            input_examples=json.dumps(tool.metadata_json.get("input_examples", [])),
            last_updated=tool.updated_at.isoformat()
        )
        tools_to_sync.append(tool_node)

    # Batch ingest to Neo4j
    for tool_node in tools_to_sync:
        episode = create_episode_payload(
            episode_type="tool_registration",
            entity=tool_node,
            store_id=0,  # System-level entity
            correlation_id=f"tool-sync-{tool_node.schema_hash}"
        )
        ingestor.ingest_episode(episode)

    logger.info(f"Synced {len(tools_to_sync)} tool nodes to Neo4j")
```

```python
# scripts/graph/init_tool_search_index.py

from src.core.graphiti_client import get_graphiti_client

def create_tool_vector_index():
    """Create vector index on ToolNode.embedding for semantic search."""
    client = get_graphiti_client()
    if not client:
        print("Neo4j not available, skipping index creation")
        return

    # Create vector index
    client.query("""
        CREATE VECTOR INDEX tool_embedding_index IF NOT EXISTS
        FOR (n:Tool)
        ON n.embedding
        OPTIONS {
            indexConfig: {
                `vector.dimensions`: 384,
                `vector.similarity_function`: 'cosine'
            }
        }
    """)

    print("Created tool_embedding_index")

if __name__ == "__main__":
    create_tool_vector_index()
```
