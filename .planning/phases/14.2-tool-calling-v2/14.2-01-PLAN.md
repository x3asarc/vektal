## Goal

Add `input_examples` field to all existing MCP tool schemas and assistant tool registry entries to improve tool-calling accuracy from ~72% to ~90% for complex nested parameters.

**Zero risk**: This is additive metadata only. No code behavior changes, no API surface changes.

---

## Background

Research shows that JSON schema alone tells an LLM "what's valid" but not "what's smart." Input examples teach conventions and practical usage patterns. For tools with nested parameters, conditional logic, or format-specific inputs (dates, file paths, queries), examples can double accuracy.

**Source**: [Anthropic Tool Calling Best Practices](https://platform.claude.com/docs/en/agents-and-tools/tool-use/implement-tool-use) - "Tool examples teach conventions... can be the difference between 60% and 90% accuracy"

### Step 4: Unit Tests

Create test to validate examples exist and match schema:

```python
# tests/graph/test_mcp_tool_examples.py

def test_all_mcp_tools_have_examples():
    """Verify all MCP tools have input_examples with valid structure."""
    from src.graph.mcp_server import list_tool_contracts

    tools = list_tool_contracts()
    assert len(tools) == 3, "Expected 3 MCP tools"

    for tool in tools:
        assert "input_examples" in tool, f"{tool['name']} missing input_examples"
        examples = tool["input_examples"]
        assert len(examples) >= 2, f"{tool['name']} needs at least 2 examples"

        # Validate examples match schema
        schema = tool["inputSchema"]
        for example in examples:
            # Check all required params present
            for required_param in schema.get("required", []):
                assert required_param in example, \
                    f"{tool['name']} example missing required param: {required_param}"


def test_assistant_tools_examples_propagate():
    """Verify tool_projection includes input_examples from metadata_json."""
    from src.assistant.tool_projection import project_tools_for_tier

    tools = project_tools_for_tier(tier=2, store_id=1, user_id=1)

    for tool in tools:
        if "input_examples" in tool:
            assert isinstance(tool["input_examples"], list)
            assert len(tool["input_examples"]) > 0
```

**File created**: `tests/graph/test_mcp_tool_examples.py`
**LOC**: ~40 lines

## Edge Cases & Validation

1. **Empty examples**: Fail test if `input_examples` is `[]`
2. **Schema mismatch**: Examples must include all `required` parameters from `inputSchema`
3. **Type mismatches**: Example values should match schema types (string/int/object)
4. **Edge case coverage**: Include at least one "unusual" example per tool:
   - Multi-file query for `query_graph`
   - `depth=3` for `get_dependencies` (max allowed)
   - Long natural language query for `retrieve_intent`
