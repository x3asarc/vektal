## Goal

Add **programmatic batch operations** to collapse multi-round-trip flows into single MCP calls:

1. `batch_query` - Execute multiple graph queries in one call
2. `batch_dependencies` - Get dependencies for multiple files in one call

**Token savings target**: 30-50% on multi-entity Tier 3 flows (e.g., 20-product resolution cycles)

---

## Background

**Tool Calling 2.0 Pattern**: "Programmatic Execution" - Run loops and conditionals in code execution environment, call tools there, return only aggregated results.

**Current bottleneck**:
- Tier 3 enrichment resolving 20 products calls `retrieve_intent` 20 times
- Each call: LLM inference pass + tool roundtrip (~500ms latency)
- Total: 20 × 500ms = 10 seconds just in round-trip overhead
- Intermediate results pile up in context (20 × ~200 tokens = 4,000 tokens)

**After batching**:
- 1 call: `batch_query(queries=["intent for SKU1", ..., "intent for SKU20"])`
- Single LLM inference pass + 1 tool roundtrip (~800ms latency)
- Aggregated result: ~1,500 tokens (vs 4,000 tokens)
- **Latency reduction**: 10s → 0.8s (12.5× faster)
- **Token reduction**: 4,000 → 1,500 (62% savings)

**Source**: [Anthropic Programmatic Tool Calling](https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling) - "Run loops in code environment, not LLM inference"

### Step 2: Implement `batch_query` Handler

```python
# src/graph/mcp_server.py

async def call_tool(name: str, arguments: dict) -> dict:
    # ... existing handlers ...

    if name == "batch_query":
        return await _batch_query_handler(
            queries=arguments["queries"],
            aggregate_mode=arguments.get("aggregate_mode", "separate")
        )


async def _batch_query_handler(queries: list[str], aggregate_mode: str) -> dict:
    """Execute multiple graph queries, return aggregated results."""
    from src.graph.query_interface import execute_natural_language_query
    import asyncio

    results = []
    errors = []

    # Execute queries concurrently
    async def _run_query(query: str, index: int):
        try:
            result = await execute_natural_language_query(query)
            return {"index": index, "query": query, "result": result, "status": "success"}
        except Exception as e:
            return {"index": index, "query": query, "error": str(e), "status": "error"}

    # Run all queries in parallel
    tasks = [_run_query(q, i) for i, q in enumerate(queries)]
    responses = await asyncio.gather(*tasks, return_exceptions=True)

    # Separate successes and failures
    for response in responses:
        if isinstance(response, Exception):
            errors.append({"error": str(response)})
        elif response["status"] == "success":
            results.append(response)
        else:
            errors.append(response)

    # Aggregate based on mode
    if aggregate_mode == "merged":
        # Combine all results into single graph structure
        merged_nodes = {}
        merged_edges = []

        for r in results:
            for node in r["result"].get("nodes", []):
                merged_nodes[node["id"]] = node
            merged_edges.extend(r["result"].get("edges", []))

        return {
            "mode": "merged",
            "total_queries": len(queries),
            "successful": len(results),
            "failed": len(errors),
            "graph": {
                "nodes": list(merged_nodes.values()),
                "edges": merged_edges
            },
            "errors": errors if errors else None
        }

    # Separate mode: return per-query results
    return {
        "mode": "separate",
        "total_queries": len(queries),
        "successful": len(results),
        "failed": len(errors),
        "results": results,
        "errors": errors if errors else None
    }
```

### Step 4: Implement `batch_dependencies` Handler

```python
# src/graph/mcp_server.py

async def call_tool(name: str, arguments: dict) -> dict:
    # ... existing handlers ...

    if name == "batch_dependencies":
        return await _batch_dependencies_handler(
            file_paths=arguments["file_paths"],
            direction=arguments.get("direction", "both"),
            depth=arguments.get("depth", 2)
        )


async def _batch_dependencies_handler(file_paths: list[str], direction: str, depth: int) -> dict:
    """Get dependencies for multiple files, return combined map."""
    from src.graph.query_interface import get_file_dependencies
    import asyncio

    results = {}
    errors = []

    # Execute dependency queries concurrently
    async def _get_deps(file_path: str):
        try:
            deps = await get_file_dependencies(file_path, direction, depth)
            return {"file": file_path, "dependencies": deps, "status": "success"}
        except Exception as e:
            return {"file": file_path, "error": str(e), "status": "error"}

    # Run all queries in parallel
    tasks = [_get_deps(fp) for fp in file_paths]
    responses = await asyncio.gather(*tasks, return_exceptions=True)

    # Build combined dependency map
    all_nodes = {}
    all_edges = []

    for response in responses:
        if isinstance(response, Exception):
            errors.append({"error": str(response)})
        elif response["status"] == "success":
            deps = response["dependencies"]
            for node in deps.get("nodes", []):
                all_nodes[node["path"]] = node
            all_edges.extend(deps.get("edges", []))
        else:
            errors.append(response)

    return {
        "total_files": len(file_paths),
        "successful": len(file_paths) - len(errors),
        "failed": len(errors),
        "dependency_map": {
            "nodes": list(all_nodes.values()),
            "edges": all_edges
        },
        "errors": errors if errors else None
    }
```

### Token Savings Test

```python
# tests/integration/test_batch_token_savings.py

def test_batch_query_token_savings():
    """Measure token reduction vs sequential calls."""
    from src.graph.mcp_server import call_tool
    import json

    queries = [f"conventions for SKU-{i:03d}" for i in range(10)]

    # Measure tokens for sequential calls
    sequential_tokens = 0
    for query in queries:
        result = call_tool("query_graph", {"query": query})
        sequential_tokens += len(json.dumps(result))

    # Measure tokens for batch call
    batch_result = call_tool("batch_query", {"queries": queries})
    batch_tokens = len(json.dumps(batch_result))

    # Assert 30% reduction
    savings = (sequential_tokens - batch_tokens) / sequential_tokens
    assert savings >= 0.30, f"Only {savings:.1%} savings, expected ≥30%"
```

## Edge Cases & Validation

1. **Empty queries array**: Return error "queries array cannot be empty"
2. **Single query**: Valid; no different from `query_graph`
3. **>50 queries**: Reject with error "max 50 queries per batch"
4. **All queries fail**: Return empty results + all errors (not fail entire call)
5. **Timeout handling**: Individual queries timeout at 5s; batch timeout at 30s
