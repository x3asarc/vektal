# Phase 02.2 Plan 01: Enrichment Module Foundation

Attribute extraction and quality scoring for German craft supply products

## Objective

Create enrichment module foundation with attribute extraction and quality scoring. Migrate core extraction logic from side-project/src/data_pipeline/ to src/core/enrichment/, establishing the foundation for the enrichment pipeline with German-first attribute extraction.

## What Was Built

Created the enrichment module with AttributeExtractor class that extracts structured attributes from German and English product text:

1. **Module structure**
   - `src/core/enrichment/` package with extractors subpackage
   - `config.py` with COLOR_MAP (38 colors), CATEGORY_KEYWORDS (9 categories), USE_CASE_PATTERNS
   - QUALITY_THRESHOLDS for scoring, ENRICHED_FIELDS for tracking

2. **AttributeExtractor class**
   - `extract_from_title()`: Extracts color, size, unit, material, category from titles
   - `extract_from_description()`: Extracts use cases and feature flags (waterproof, washable, non-toxic)
   - `extract_all()`: Combined extraction from both title and description
   - `calculate_quality_score()`: 0-100 score based on completeness formula

3. **Pattern support**
   - Colors: German (rot, grün, blau) and English (red, green, blue) normalized to German
   - Special colors: jade → Jade Grün, ruby → Ruby Rot
   - Sizes: 20ml, 50g, 14x14cm, A4, decimal formats (0,5l)
   - Materials: Acryl, Textil, Harz, Öl, Papier (compound word support)
   - Categories: 9 categories (Farbe, Kleber, Folie, Papier, Werkzeug, Verzierung, Harz, Stempel, Schablone)

4. **Quality scoring**
   - Description quality: 40 points (>100 chars = 40, >50 = 30, >20 = 15)
   - Structured data: 30 points (color +10, size +10, material +10)
   - Categorization: 20 points (category +10, product type +10)
   - Tags: 10 points (tags present +10)
   - High-quality products (all fields) score 85+
   - Low-quality products (missing data) score <50

5. **Comprehensive testing**
   - 34 unit tests covering all extraction patterns
   - Tests for German/English colors, sizes, materials, categories
   - Quality score edge cases and bounds testing
   - All tests pass

## Decisions Made

1. **German-first design**: All colors normalized to German (Rot not Red) for German market SEO
2. **Pattern priority ordering**: Specific patterns (dimensions, paper formats) before general (simple sizes)
3. **Compound word support**: Partial word boundaries for German compound words (Epoxidharz)
4. **Quality score formula**: 40/30/20/10 weighting (description > structured data > categorization > tags)

## Known Limitations

1. **Single color extraction**: Only extracts first color found (multi-color products return one)
2. **No context-aware extraction**: "Gold" could be color or material, no disambiguation
3. **Fixed regex patterns**: New size formats require code changes, not configurable
4. **German-only use cases**: USE_CASE_PATTERNS only in German (Decoupage, Schmuckherstellung, etc.)
