## Decisions Made

### 1. Vendor Auto-Detection
**Decision:** Detect vendor from product['vendor'] field, normalize to slug format

**Rationale:** Products already have vendor field from scraping. Normalizing to slug (lowercase, replace non-alphanumeric with underscores) makes finding config/vendors/{slug}.yaml automatic.

**Implementation:**
```python
detect_vendor_from_product({'vendor': 'ITD Collection'})  # → 'itd_collection'
detect_vendor_from_product({'vendor': 'Pentart Hungary Kft.'})  # → 'pentart_hungary_kft'
```

**Impact:** Zero-config vendor enrichment. If YAML exists, rules apply automatically.

### 2. Conditional OR Support
**Decision:** Support OR conditions in tagging rules

**Rationale:** Common pattern: "If title contains 'vintage' OR 'retro', add 'style:vintage' tag". Without OR, requires duplicate rules.

**YAML Example:**
```yaml
conditional:
  - condition: "title contains 'vintage' OR title contains 'retro'"
    add_tags:
      - "style:vintage"
```

**Impact:** More concise vendor YAML configs. Reduces duplication from ~5 rules to 1 rule with OR.

### 3. Vendor Rules After Extraction
**Decision:** Apply vendor rules as Step 1.5 (after attribute extraction)

**Rationale:** Vendor rules may reference extracted attributes (e.g., `size == 'A4'`). Must run after extraction to have extracted_size, extracted_color, etc.

**Alternatives considered:**
- Before extraction: Wouldn't have extracted attributes for conditions
- After all enrichment: Too late, AI generation might overwrite vendor-specific content

**Impact:** Vendor conditional rules can use extracted fields. Example: "If size is A4, add to 'Rice Paper A4' collection".

## Key Design Patterns

### 1. Dataclass-Based Configuration
**Pattern:** Use Python dataclasses for structured YAML sections
```python
@dataclass
class EnrichmentKeywords:
    primary: List[str] = field(default_factory=list)
    secondary: List[str] = field(default_factory=list)
    ...
```
**Benefit:** Type safety, default values, clean structure

### 2. Lazy Config Loading
**Pattern:** Load config only when path/slug provided
```python
if self.config_path and Path(self.config_path).exists():
    self._load_config()
```
**Benefit:** Graceful handling when vendor YAML doesn't exist

### 3. Pipeline Step Extension
**Pattern:** Add new step without breaking existing flow
```python
# Step 1: Extract attributes
products = self._step_extract_attributes(products)
# Step 1.5: Apply vendor rules (NEW - inserted)
products = self._step_apply_vendor_rules(products)
# Step 2: Apply templates (EXISTING)
```
**Benefit:** Non-breaking addition. Can skip vendor rules if no config.
