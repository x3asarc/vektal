    2. Create docs/SCRAPER_STRATEGY.md:

       ```markdown
       # Scraper Strategy: Python vs JavaScript

       ## Context

       This project has two scraping implementations:
       - Python (BeautifulSoup/Selenium): src/core/scrape_engine.py, image_scraper.py
       - JavaScript (Playwright): universal_vendor_scraper/

       This document clarifies when to use each and how they integrate.

       ## Decision

       **Primary scraper: Python** (src/core/scrape_engine.py)
       **Secondary scraper: JavaScript** (for JavaScript-heavy sites)

       ## Rationale

       ### Use Python When:
       - Static HTML pages (BeautifulSoup is faster)
       - Integration with existing pipeline (pipeline.py)
       - Simple form submissions
       - Majority of vendor sites

       ### Use JavaScript When:
       - Single-page applications (SPA)
       - Sites requiring JavaScript execution to render content
       - Complex user interactions (scroll-to-load, dynamic filters)
       - Anti-scraping measures requiring browser fingerprinting

       ## Current Vendor Assignments

       | Vendor | Scraper | Reason |
       |--------|---------|--------|
       | Pentart | Python | Static HTML |
       | ITD | JavaScript | Dynamic pricing |
       | Aistcraft | JavaScript | SPA |
       | PaperDesigns | Python | Static HTML |
       | [others] | TBD | Analysis needed |

       ## Integration Pattern

       JavaScript scrapers output JSON that Python pipeline consumes:

       ```
       JS Scraper -> JSON file -> Python pipeline.py -> Shopify
       ```

       1. JS scraper writes to results/scrape_output.json
       2. Python reads JSON as input to pipeline
       3. Single source of truth for Shopify updates

       ## Configuration

       Both scrapers read from same vendor config:
       - config/vendors.yaml (canonical)

       Python: vendor_config.py loads YAML
       JavaScript: reads YAML via yaml package

       ## Adding New Vendors

       1. Analyze site (static vs dynamic)
       2. Choose scraper based on criteria above
       3. Add vendor config to config/vendors.yaml
       4. Implement strategy in appropriate scraper
       5. Test with --dry-run before production

       ## Future Direction

       Phase 2 will unify scraper interface:
       - Single Python entry point
       - JavaScript called as subprocess when needed
       - Unified output format
       - Shared vendor configuration

       ## Open Questions

       - Should we migrate all to Playwright (JavaScript) for consistency?
       - Performance comparison needed
       - Memory usage of browser instances

       See Phase 2 roadmap for resolution timeline.
       ```
